This is a list of what versions of the DV library dependencies are
currenly in use.  Because DV hasn't historically had the versions
in the JAR names, this is a good housekeeping practice in its place.

Bean Scripting Framework           - bsf.jar             - 2.4.0	http://jakarta.apache.org/site/downloads/downloads_bsf.cgi
iText PDF generator                - iText.jar           - 2.0.1	http://www.lowagie.com/iText/download.html
jCalendar date picker widget       - jcalendar.jar       - 1.3.2	http://www.toedter.com/en/jcalendar/
jRuby                              - jruby.jar           - 0.9.8	http://dist.codehaus.org/jruby/
Jakarta Commons Logging            - commons-logging.jar - 1.1		http://commons.apache.org/logging/
ASM Byte Code Generation Framework - asm.jar             - 2.2.3        http://forge.objectweb.org/projects/asm/
jUnit                              - junit.jar           - 3.8.1	http://sourceforge.net/project/showfiles.php?group_id=15278
POI (Jakarta MS Office/Excel API)  - poi.jar             - 3.0.alpha3.20061212	http://www.apache.org/dyn/closer.cgi/poi/
Apache Derby Database              - derby.jar           - 10.3.2.1	http://db.apache.org/derby/derby_downloads.html
SQLite JDBC Driver                 - sqlitejdbc.jar      - v044/3.5.8	http://www.zentus.com/sqlitejdbc/

Last updated 1/23/2008 by Frank W. Zammetti
URLs to apparently official download locations added 2008-03-23 by Jonathan Marsden
Derby and SQLite updated to current versions 2008-04-18 by Jonathan Marsden

* Note: Derby is only needed if you intend to run the sample report
  named live_database_report_example.xml in the /examples directory.  
  It -IS NOT- needed to run DataVision in general!

* Note: SQLite JDBC driver (and DLL) are only needed if you intend
  to run the sample report named sqlite_example.xml in the
  /examples directory.  It -IS NOT- needed to run DataVision in
  general!

* Liquid Look and Feel (liquidlnf.jar) has been removed
